## 1. Resumen Ejecutivo

Este documento deja establecido el perfil comercial del prospecto que se debe buscar con el proyecto `prospectos-leadmaster-local`.

El proyecto no busca prospectos para clientes finales, sino **empresas que podrían comprar el servicio LeadMaster**. Por lo tanto, el objetivo es detectar anunciantes activos, especialmente en Google Ads o Bing Ads, que ya están invirtiendo dinero para conseguir clientes y que podrían necesitar un sistema de prospección comercial calificada.

La hipótesis central es:

> Si una empresa B2B ya paga anuncios para conseguir consultas, probablemente tiene necesidad comercial, presupuesto mínimo y un problema concreto de captación. LeadMaster puede ofrecerle oportunidades comerciales calificadas, no simples bases de datos.

---

## 2. Distinción conceptual principal

Es fundamental separar dos tipos de prospectos:

| Concepto | Definición |
|---|---|
| **Prospecto-LeadMaster** | Empresa a la que nosotros queremos venderle LeadMaster. |
| **Prospecto para el cliente** | Contacto calificado que LeadMaster le entrega a un cliente que contrató el servicio. |

Este documento se refiere exclusivamente al primer caso: **prospectos para vender LeadMaster**.

---

## 3. Definición del prospecto ideal

El prospecto ideal para LeadMaster es:

> **Una empresa B2B, preferentemente argentina, que ya invierte en Google Ads o Bing Ads, vende productos o servicios de ticket medio/alto, necesita conseguir clientes nuevos y no tiene un sistema propio sólido de prospección comercial calificada.**

No buscamos cualquier anunciante. Buscamos empresas donde una nueva oportunidad comercial tenga valor económico suficiente para justificar el servicio.

---

## 4. Criterios principales de selección

### 4.1. Debe ser preferentemente B2B

Priorizar empresas que venden a otras empresas, instituciones, comercios, industrias o profesionales.

Ejemplos:

- proveedores industriales;
- distribuidores mayoristas;
- empresas de servicios técnicos;
- proveedores para agro;
- proveedores para minería;
- proveedores para energía;
- importadores;
- fabricantes que venden a empresas;
- empresas de equipamiento profesional;
- servicios especializados para comercios, industrias o instituciones.

### 4.2. Debe tener ticket medio o alto

LeadMaster se justifica mejor cuando una venta nueva tiene valor relevante.

Rubros adecuados:

- maquinaria;
- insumos técnicos;
- equipamiento profesional;
- servicios industriales;
- construcción especializada;
- servicios para empresas;
- soluciones para agro, minería o energía;
- productos importados;
- mantenimiento técnico;
- instalaciones;
- servicios tercerizados.

### 4.3. Debe estar pagando anuncios

Este es un criterio estratégico.

Una empresa que ya paga Google Ads o Bing Ads muestra tres señales importantes:

1. necesita clientes;
2. tiene presupuesto comercial;
3. ya reconoce el valor de invertir para generar demanda.

Por eso el proyecto `prospectos-leadmaster-local` debe enfocarse en capturar landing pages de anuncios patrocinados y no listados orgánicos genéricos.

### 4.4. Debe necesitar oportunidades comerciales, no una lista

LeadMaster no debe presentarse como un vendedor de bases de datos.

La propuesta debe centrarse en:

- contactos que respondieron;
- contactos que pidieron información;
- contactos que solicitaron precio o catálogo;
- empresas que manifestaron necesidad;
- oportunidades comerciales revisables por un vendedor.

La promesa comercial no es “te damos 500 contactos”, sino:

> “Te acercamos oportunidades comerciales calificadas con intención manifiesta.”

### 4.5. Debe carecer de una maquinaria propia de prospección

El prospecto ideal puede tener web, anuncios, WhatsApp, formularios o vendedores, pero no suele tener un proceso sistemático para:

- detectar prospectos;
- segmentarlos;
- contactarlos;
- registrar respuestas;
- hacer seguimiento;
- clasificar interés;
- entregar oportunidades listas para gestión comercial.

Ese vacío es el espacio de LeadMaster.

## 5. Rubros priorizados

### 5.1. Alta prioridad

| Sector | Motivo |
|---|---|
| Importadores | Venden productos de mayor valor, necesitan clientes B2B y no dependen exclusivamente de la industria local. |
| Proveedores agropecuarios | El agro mantiene demanda de insumos, servicios y equipamiento. |
| Proveedores para minería | Ticket alto, cadenas de compra complejas y necesidad de proveedores especializados. |
| Proveedores para energía | Buen encaje para contratistas, equipamiento, mantenimiento y servicios técnicos. |
| Empresas industriales B2B | Buen modelo de referencia para LeadMaster, especialmente si ya usan Google Ads. |
| Servicios técnicos especializados | Suelen depender de presupuestos y consultas calificadas. |
| Distribuidores mayoristas | Necesitan ampliar cartera de clientes comerciales. |
| Equipamiento profesional | Venta consultiva, ticket medio/alto y clientes identificables. |

### 5.2. Prioridad media

| Sector | Observación |
|---|---|
| Instituciones educativas privadas | Pueden necesitar captación, pero no siempre encajan en B2B. |
| Comercios con ticket alto | Válidos si venden productos caros o presupuestables. |
| Profesionales o empresas de servicios | Válidos si el valor por cliente justifica la prospección. |
| Empresas de marketing o software | Analizar con cuidado: podrían resolver internamente parte del problema. |

### 5.3. Baja prioridad o evitar

| Sector | Motivo |
|---|---|
| Restaurantes, bares y cafeterías | B2C, ticket bajo y dinámica más dependiente de redes sociales. |
| E-commerce minorista genérico | Competencia alta, ticket variable y menor venta consultiva. |
| Negocios muy chicos | Probable baja capacidad de pago. |
| Servicios de bajo ticket | No justifican el costo de prospección. |
| Rubros donde una venta vale poco | LeadMaster necesita que cada oportunidad tenga valor económico suficiente. |

## 6. Fórmula operativa

Un prospecto válido para LeadMaster debería cumplir la mayor cantidad posible de estas condiciones:

```text
Prospecto-LeadMaster válido =
Empresa B2B
+ ticket medio/alto
+ presencia activa en Google Ads o Bing Ads
+ necesidad visible de conseguir clientes
+ capacidad de pago
+ canal comercial activo
+ ausencia de sistema propio de prospección calificada
```

---

## 7. Sistema inicial de calificación

Cada anunciante detectado puede puntuarse con el siguiente esquema:

| Criterio                                                      | Puntaje |
| ------------------------------------------------------------- | ------: |
| Es B2B claro                                                  |      +3 |
| Vende producto o servicio de ticket alto                      |      +3 |
| Está anunciando en Google o Bing                              |      +2 |
| Tiene WhatsApp, formulario o llamada a la acción visible      |      +2 |
| Tiene landing orientada a captar consultas                    |      +2 |
| Atiende empresas, industrias, instituciones o profesionales   |      +2 |
| Tiene alcance regional, nacional o sectorial amplio           |      +1 |
| Parece negocio chico de bajo ticket                           |      -3 |
| Es B2C puro                                                   |      -3 |
| Es gastronómico o minorista simple                            |      -2 |
| Es software, marketing o agencia con capacidad técnica propia |      -1 |

Clasificación sugerida:

```text
0 a 3 puntos: descartar
4 a 6 puntos: revisar manualmente
7 a 10 puntos: prospecto válido
11 puntos o más: prospecto prioritario
```

## 8. Flujo recomendado del proyecto

El flujo operativo de `prospectos-leadmaster-local` debería ser:

```text
Google Ads / Bing Ads
→ detectar anunciantes activos
→ capturar landing page
→ extraer texto por OCR
→ identificar rubro y señales B2B
→ puntuar prospecto
→ guardar en MySQL
→ revisar manualmente
→ contactar con propuesta LeadMaster
```

El proyecto debe funcionar como radar de mercado, no como módulo operativo de campañas.

---

## 9. Relación con LeadMaster Central Hub

`prospectos-leadmaster-local` debe considerarse un módulo externo de captación de posibles clientes para LeadMaster.

No debería escribir directamente sobre tablas operativas de `central-hub` sin una integración definida.

La integración recomendable sería:

1. mantener una tabla propia de prospectos LeadMaster;
2. calificar manual o automáticamente esos registros;
3. enviar solo los prospectos validados al flujo comercial de LeadMaster mediante un endpoint formal.

Ejemplo conceptual:

```http
POST /api/prospectos-leadmaster/importar
```

## 11. Mensaje comercial base

La propuesta comercial para estos prospectos no debe formularse como venta de software ni como venta de base de datos.

Mensaje conceptual:

> “Vimos que están invirtiendo en captar clientes. Desde LeadMaster podemos ayudarlos a conseguir oportunidades comerciales calificadas: contactos que manifiestan interés, piden información, solicitan precio o requieren asesoramiento.”

El eje del discurso debe ser:

* bajar costo de captación;
* complementar Google Ads;
* generar oportunidades B2B;
* evitar que el equipo comercial pierda tiempo con contactos fríos;
* entregar interés manifiesto y seguimiento.

---

## 12. Conclusión

El perfil más fuerte para LeadMaster es una empresa tipo **Mandarinsa**: B2B, con producto o servicio concreto, búsqueda activa de clientes, inversión en Google Ads y posibilidad de valorar un canal complementario de prospección.

La estrategia no debe dispersarse en negocios genéricos de bajo ticket. Debe enfocarse en anunciantes B2B donde una oportunidad comercial pueda justificar ampliamente el costo del servicio.
